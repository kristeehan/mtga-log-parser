# Plan: v1.0.0 — Format-Agnostic Parser

## Context

The current library is hardcoded to accept only Bo3 constructed matches (`Traditional_*` / `Constructed_BestOf3`). All other formats — Bo1, limited, bot matches — are silently discarded. This is a major-version redesign that makes the parser format-agnostic by default, with opt-in filtering via a `matchFilter` predicate in `ParseConfig`.

The internal collection logic (`gameDataCollector`, `boardStateCollector`, `tryExtractGameState`) is already fully format-agnostic. The gating is entirely in two small functions (`isBo3Constructed`, `isConstructedEvent`) and one call site in `handleMatchStart` — so the changes are minimal and well-contained.

---

## Breaking Changes

| What | Old behavior | New behavior |
|---|---|---|
| No `matchFilter` provided | Returns only Bo3 constructed matches | Returns all matches |
| `deckUsages` | Tracks only constructed formats | Tracks all formats |
| `isBo3Constructed` | Internal function | Removed; replaced by `MatchFilters.bo3Constructed` |
| `isConstructedEvent` | Internal function | Removed |

Callers who relied on the old default must add `matchFilter: MatchFilters.bo3Constructed` to their config.

### 3. Add exported `MatchFilters` presets — `src/logParser.ts`

Add near the top of the file, after imports:

```typescript
export const MatchFilters = {
  /** Traditional_ prefix (ranked + seasonal events) and Constructed_BestOf3 only */
  bo3Constructed: (eventId: string): boolean =>
    eventId.startsWith('Traditional_') || eventId === 'Constructed_BestOf3',

  /** All constructed formats — excludes draft, sealed, and jump-in */
  constructed: (eventId: string): boolean =>
    Boolean(eventId) && !/(Draft|Sealed|Jump_?In)/i.test(eventId),

  /** All matches — the default when matchFilter is omitted */
  all: (): boolean => true,
} as const;
```

## What Does NOT Change

- `Match`, `GameResult`, `MatchResult`, `MatchUpdate` — `game2`/`game3` are already nullable; Bo1 matches set only `game1`.
- `GameSnapshot`, `TurnSnapshot`, `BoardCard` — format-agnostic already.
- `computeMatchResult` — already handles single-game Bo1 correctly (returns `'Win'`/`'Loss'` when only `g1` is non-null).
- `handleMatchEnd` — reads `finalMatchResult.resultList`, which is format-agnostic.
- `tryExtractGameState`, `gameDataCollector`, `boardStateCollector` — all format-agnostic.
- `resolveColors` callback — unchanged.
- `ParseResult` shape — unchanged.
